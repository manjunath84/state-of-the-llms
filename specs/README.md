# specs/ — the project constitution (SDD)

This directory is **canonical**. Read it before changing scope. Per ADR 0002
(`../docs/decisions/0002-real-usage-derived-offline.md` family — see also the
SSM-PDFTool origin), there is **no `PLAN.md`**: the spec governs, and the
per-feature plan is a transient task list.

Adapted from the SSM-PDFTool / SSM-Transcriber SDD constitution; the **goal**
comes from the Gen Academy Week 1 handout, not from SSM.

| File | What it holds |
|---|---|
| `mission.md` | Why this exists, audience, in/out of scope |
| `tech-stack.md` | The layers + the 8 binding guardrails (review-blockers) |
| `roadmap.md` | The 5-session schedule as phases, with status |
| `REQUIREMENTS_TEMPLATE.md` | Copy this when starting a new feature spec |
| `2026-05-30-state-of-the-llms/` | This feature's requirements / plan / validation |

The full design narrative (the source these files are distilled from) lives at
`../docs/superpowers/specs/2026-05-30-state-of-the-llms-design.md`.
